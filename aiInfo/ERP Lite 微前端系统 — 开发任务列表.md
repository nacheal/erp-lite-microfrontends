## 阶段一：工程基础搭建

> 目标：建立 monorepo 骨架，完成所有应用的本地开发环境，所有应用能独立跑起来。

### 1.1 Monorepo 初始化

- [ ] **T-001** 初始化 Git 仓库，配置 `.gitignore`
- [ ] **T-002** 安装 pnpm，初始化 `pnpm-workspace.yaml`，定义 `packages/*` 工作区
- [ ] **T-003** 根目录 `package.json` 配置公共脚本（`dev:all`、`build:all`、`lint:all`）
- [ ] **T-004** 配置根目录 TypeScript（`tsconfig.base.json`），各子包继承

### 1.2 代码规范配置

- [ ] **T-005** 配置 ESLint（支持 React、Vue、TypeScript 规则集）`依赖 T-002`
- [ ] **T-006** 配置 Prettier，与 ESLint 对齐，避免规则冲突
- [ ] **T-007** 配置 Stylelint（CSS / SCSS 规范）
- [ ] **T-008** 配置 husky + lint-staged，在 `pre-commit` 阶段自动执行检查 `依赖 T-005 T-006`
- [ ] **T-009** 配置 commitlint，统一 commit message 格式（feat/fix/chore 等）

### 1.3 创建各应用骨架

- [ ] **T-010** 创建 `packages/shell-app`，使用 Create React App 或 Vite 初始化，配置 Webpack 5 `依赖 T-002`
- [ ] **T-011** 创建 `packages/app-user`，React 18 + TypeScript + Webpack 5 `依赖 T-002`
- [ ] **T-012** 创建 `packages/app-product`，Vue 3 + TypeScript + Vite 5 `依赖 T-002`
- [ ] **T-013** 创建 `packages/app-order`，React 18 + TypeScript + Webpack 5 `依赖 T-002`
- [ ] **T-014** 创建 `packages/app-dashboard`，原生 JS + Vite 5 `依赖 T-002`

### 1.4 共享包初始化

- [ ] **T-015** 创建 `shared/types`，定义跨应用共享的 TypeScript 接口（`GlobalState`、`UserInfo`、`MicroAppConfig` 等）`依赖 T-004`
- [ ] **T-016** 创建 `shared/utils`，初始化纯函数工具目录（格式化、校验等），**禁止引入框架依赖** `依赖 T-004`

### 1.5 本地开发验证

- [ ] **T-017** 验证各应用可独立启动（按端口规划：3000 / 3001 / 3002 / 3003 / 3004）`依赖 T-010 ~ T-014`
- [ ] **T-018** 配置 Mock Server（端口 4000），使用 `json-server` 或 `msw` 提供本地 API `依赖 T-002`

## 阶段二：主应用开发

> 目标：完成 Shell App 的完整骨架，包括布局、路由、登录鉴权、全局状态初始化，为接入子应用做好准备。

### 2.1 整体布局

- [ ] **T-019** 实现主应用整体布局组件（顶部导航栏 + 左侧菜单 + 内容区）`依赖 T-010`
- [ ] **T-020** 实现左侧菜单组件，菜单项与路由联动，支持收起/展开
- [ ] **T-021** 实现顶部导航栏组件（Logo、用户信息展示、退出登录入口）
- [ ] **T-022** 实现响应式布局，窗口缩小时菜单自动折叠

### 2.2 路由配置

- [ ] **T-023** 配置 React Router 6，定义主应用一级路由（`/login`、`/`、子应用挂载点）`依赖 T-019`
- [ ] **T-024** 实现路由守卫（未登录重定向到 `/login`，已登录重定向到 `/dashboard`）
- [ ] **T-025** 实现 404 页面

### 2.3 登录与鉴权

- [ ] **T-026** 实现登录页 UI（账号密码表单，含表单校验）`依赖 T-023`
- [ ] **T-027** 实现登录接口调用，登录成功后存储 token（内存 + sessionStorage）
- [ ] **T-028** 实现退出登录逻辑（清空 token，重置全局状态，跳转登录页）
- [ ] **T-029** 实现 token 过期自动刷新或过期跳转登录页逻辑

### 2.4 全局状态初始化

- [ ] **T-030** 安装 qiankun，初始化 `initGlobalState`，定义 `GlobalState` 初始值 `依赖 T-015`
- [ ] **T-031** 登录成功后将 `userInfo`、`token`、`permissions` 写入全局状态
- [ ] **T-032** 实现全局状态变更监听，处理子应用发出的 `action`（如 `LOGOUT`、`NAVIGATE_*`）

### 2.5 权限控制

- [ ] **T-033** 实现权限工具函数（`hasPermission(code: string): boolean`）`依赖 T-031`
- [ ] **T-034** 实现菜单项的权限过滤（无权限的菜单项不显示）

## 阶段七：性能优化

> 目标：达到性能基准目标（FCP < 1.5s，子应用首次激活 < 1s，切换已缓存子应用 < 200ms）。

### 7.1 Bundle 优化

- [ ] **T-106** 安装 `webpack-bundle-analyzer`，分析各应用 bundle 构成，找出超大依赖 `依赖 阶段四完成`
- [ ] **T-107** 配置 externals，将 react / react-dom / vue / axios 从子应用 bundle 排除，改为 CDN 引入
- [ ] **T-108** 配置代码分割（`optimization.splitChunks`），将路由页面按需拆分为独立 chunk
- [ ] **T-109** 验证各子应用 bundle 大小达到目标值（< 500KB gzip），不达标则继续优化

### 7.2 加载策略优化

- [ ] **T-110** 配置 qiankun `prefetchApps`，在主应用首屏渲染后按优先级预加载子应用
- [ ] **T-111** 验证预加载效果：第二次访问子应用时加载耗时明显缩短，记录对比数据

### 7.3 子应用缓存

- [ ] **T-112** 将高频子应用（app-user、app-order）改用 `loadMicroApp` 手动管理，实现切走后不销毁
- [ ] **T-113** 验证缓存后切换耗时 < 200ms，同时确认内存占用在可接受范围内

### 7.4 性能数据采集

- [ ] **T-114** 在子应用 `mount` / `unmount` 生命周期中添加 `performance.mark` 打点
- [ ] **T-115** 汇总优化前后的性能数据对比报告，更新性能基准目标完成情况
