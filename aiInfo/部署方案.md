## 1. 部署策略概述

### 1.1 部署原则

ERP Lite 微前端系统采用 qiankun 架构，由 1 个主应用（shell-app）和 4 个微应用组成。部署到 Vercel 遵循以下原则：

- **独立部署**：每个应用独立部署为一个 Vercel 项目，实现独立发布和版本管理
- **按需加载**：子应用通过主应用动态加载，用户访问时才加载对应模块
- **环境隔离**：生产环境、测试环境分别部署，通过环境变量区分
- **灰度发布**：支持子应用独立更新，无需主应用重新部署

### 1.2 部署拓扑

```
┌─────────────────────────────────────────────────────────────┐
│                         Vercel 平台                          │
├─────────────────────────────────────────────────────────────┤
│                                                               │
│  ┌──────────────┐      主应用                               │
│  │  shell-app   │      https://erp-lite-shell.vercel.app    │
│  └──────┬───────┘                                            │
│         │                                                     │
│         ├──────► app-user      (微应用1)                    │
│         │        https://erp-lite-user.vercel.app           │
│         │                                                     │
│         ├──────► app-product   (微应用2)                    │

## 2. Vercel 部署优势与挑战

### 2.1 优势

| 优势 | 说明 |
|------|------|
| **零配置 CI/CD** | 自动从 GitHub/GitLab 拉取代码，自动构建和部署 |
| **全球 CDN** | 自动分发到全球边缘节点，访问速度快 |
| **自动 HTTPS** | 自动配置 SSL 证书 |
| **预览部署** | 每个 PR 自动生成预览环境 |
| **环境变量管理** | 支持多环境变量配置（开发/预览/生产） |
| **性能监控** | 内置 Analytics 和 Web Vitals 监控 |
| **免费额度** | 个人项目和小团队可免费使用 |

### 2.2 挑战与应对

| 挑战 | 影响 | 应对方案 |
|------|------|----------|
| **Monorepo 构建** | Vercel 默认构建整个仓库根目录 | 使用 `vercel.json` 的 `buildCommand` 和 `outputDirectory` 指定构建路径 |
| **跨域配置** | 子应用需要被主应用跨域加载 | 在 `vercel.json` 中配置 CORS 响应头 |
| **动态路由回退** | SPA 刷新时 404 | 配置 `rewrites` 实现 History 路由回退 |
| **环境变量同步** | 5 个项目需分别配置 | 使用脚本统一管理，或使用 Vercel CLI 批量导入 |
| **构建时长限制** | 免费版构建时间有限制 | 优化构建配置，移除不必要的依赖 |

### 3.1 应用清单

| 应用名称 | 技术栈 | 构建工具 | 端口(本地) | Vercel 项目名 | 生产域名(示例) |
|---------|--------|---------|-----------|--------------|---------------|

| app-dashboard | 原生 JS | Vite 5 | 3004 | erp-lite-dashboard | erp-lite-dashboard.vercel.app |

### 3.2 依赖关系

```
主应用 (shell-app)
  ├─ 通过环境变量引用子应用 URL
  │
  ├─ VITE_APP_USER_ENTRY = https://erp-lite-user.vercel.app

  ├─ VITE_APP_ORDER_ENTRY = https://erp-lite-order.vercel.app

- **开发环境(Development)**：本地 localhost 运行，Mock Server 提供 API
- **预览环境(Preview)**：每个 Pull Request 自动部署预览版本
- **生产环境(Production)**：主分支（main）自动部署到生产环境

#### 5.1.2 修改子应用注册配置

修改 `packages/shell-app/src/micro/index.ts`：

```typescript
/**
 * 获取子应用入口地址
 * 开发环境使用 localhost，生产环境使用环境变量配置的 URL
 */
const getEntry = (localPort: number, envKey: string): string => {
  // 检查是否在 qiankun 沙箱中运行
  if (window.__POWERED_BY_QIANKUN__) {
    return `//localhost:${localPort}`;
  }

  // 生产环境从环境变量读取
  const prodEntry = import.meta.env[envKey];
  if (prodEntry) {
    return prodEntry;
  }

  // 默认使用本地开发地址
  return `//localhost:${localPort}`;
};

/**
 * 子应用配置列表
 */
const microApps: MicroApp[] = [
  {
    name: 'app-user',
    entry: getEntry(3001, 'VITE_APP_USER_ENTRY'),
    container: '#subapp-user',
    activeRule: (location: Location) => location.pathname.startsWith('/user'),
    props: getProps(),
  },
  {
    name: 'app-product',
    entry: getEntry(3002, 'VITE_APP_PRODUCT_ENTRY'),
    container: '#subapp-product',
    activeRule: (location: Location) => location.pathname.startsWith('/product'),
    props: getProps(),
  },
  {
    name: 'app-order',
    entry: getEntry(3003, 'VITE_APP_ORDER_ENTRY'),
    container: '#subapp-order',
    activeRule: (location: Location) => location.pathname.startsWith('/order'),
    props: getProps(),
  },
  {
    name: 'app-dashboard',
    entry: getEntry(3004, 'VITE_APP_DASHBOARD_ENTRY'),
    container: '#subapp-dashboard',
    activeRule: (location: Location) => location.pathname.startsWith('/dashboard'),
    props: getProps(),
  },
];
```

## 7. 部署流程

### 7.1 方式一：通过 Vercel Dashboard（推荐新手）

#### 步骤 1：创建主应用项目

1. 访问 https://vercel.com/new
2. 选择 GitHub 仓库
3. 项目配置：
   - **Project Name**: `erp-lite-shell`
   - **Framework Preset**: `Other`
   - **Root Directory**: `packages/shell-app`
   - **Build Command**: `cd ../.. && pnpm --filter shell-app build`
   - **Output Directory**: `dist`
   - **Install Command**: `cd ../.. && pnpm install`
4. 配置环境变量（参见 6.1.1）
5. 点击 **Deploy**

#### 步骤 2：创建微应用项目

重复上述步骤，分别为以下项目创建 Vercel 项目：

| 项目名 | Root Directory | Build Command |
|--------|---------------|---------------|
| erp-lite-user | packages/app-user | `cd ../.. && pnpm --filter app-user build` |
| erp-lite-product | packages/app-product | `cd ../.. && pnpm --filter app-product build` |
| erp-lite-order | packages/app-order | `cd ../.. && pnpm --filter app-order build` |
| erp-lite-dashboard | packages/app-dashboard | `cd ../.. && pnpm --filter app-dashboard build` |

#### 步骤 3：更新主应用环境变量

部署完所有微应用后，获取它们的生产域名，然后回到 shell-app 项目：

1. Settings → Environment Variables
2. 更新子应用 URL 为实际部署的域名
3. 触发重新部署（Deployments → 三点菜单 → Redeploy）

### 7.3 自动化部署脚本

创建 `scripts/deploy.sh`：

```bash
#!/bin/bash

set -e

echo "🚀 开始部署 ERP Lite 微前端系统..."

# 颜色定义
GREEN='\033[0;32m'
YELLOW='\033[1;33m'
NC='\033[0m' # No Color

# 部署微应用
echo -e "${YELLOW}📦 部署微应用...${NC}"

apps=("app-user" "app-product" "app-order" "app-dashboard")
urls=()

for app in "${apps[@]}"; do
  echo -e "${GREEN}正在部署 $app...${NC}"
  cd "packages/$app"
  url=$(vercel --prod --yes 2>&1 | grep -o 'https://[^ ]*' | head -1)
  urls+=("$url")
  echo -e "${GREEN}✅ $app 部署完成: $url${NC}"
  cd ../..
done

# 配置主应用环境变量
echo -e "${YELLOW}🔧 配置主应用环境变量...${NC}"
cd packages/shell-app

vercel env rm VITE_APP_USER_ENTRY production --yes || true
vercel env add VITE_APP_USER_ENTRY production <<< "${urls[0]}"

vercel env rm VITE_APP_PRODUCT_ENTRY production --yes || true
vercel env add VITE_APP_PRODUCT_ENTRY production <<< "${urls[1]}"

vercel env rm VITE_APP_ORDER_ENTRY production --yes || true
vercel env add VITE_APP_ORDER_ENTRY production <<< "${urls[2]}"

vercel env rm VITE_APP_DASHBOARD_ENTRY production --yes || true
vercel env add VITE_APP_DASHBOARD_ENTRY production <<< "${urls[3]}"

# 部署主应用
echo -e "${YELLOW}🎯 部署主应用...${NC}"
shell_url=$(vercel --prod --yes 2>&1 | grep -o 'https://[^ ]*' | head -1)
echo -e "${GREEN}✅ 主应用部署完成: $shell_url${NC}"

cd ../..

echo -e "${GREEN}🎉 部署完成！${NC}"
echo -e "${GREEN}主应用地址: $shell_url${NC}"
```

使用脚本部署：

```bash
chmod +x scripts/deploy.sh
./scripts/deploy.sh
```

## 10. 常见问题与解决方案

### 10.1 构建失败

**问题**：构建时报错 `pnpm: command not found`

**解决方案**：
1. 在 Vercel 项目设置中确认 `Install Command` 使用 pnpm
2. 添加 `.npmrc` 文件：
   ```
   package-manager=pnpm@8.15.0
   ```

### 10.2 子应用加载失败

**问题**：主应用部署后，子应用加载失败，控制台显示 `Failed to fetch`

**原因**：
- 子应用 URL 配置错误
- 子应用 CORS 未正确配置

**解决方案**：
1. 检查主应用环境变量中的子应用 URL 是否正确
2. 确认子应用 `vercel.json` 中的 CORS 配置
3. 检查子应用是否成功部署

### 10.3 刷新 404

**问题**：在子应用页面刷新浏览器出现 404

**原因**：Vercel 默认不支持 SPA 路由回退

**解决方案**：
确保 `vercel.json` 中配置了 `rewrites`：
```json
{
  "rewrites": [
    {
      "source": "/(.*)",
      "destination": "/index.html"
    }
  ]
}
```

### 10.4 环境变量未生效

**问题**：修改了环境变量但应用中仍使用旧值

**原因**：Vercel 缓存了构建结果

**解决方案**：
1. Deployments → 最新部署 → 三点菜单 → **Redeploy**
2. 勾选 **Use existing Build Cache** 取消勾选
3. 点击 **Redeploy**

### 10.5 跨域问题

**问题**：控制台报错 `CORS policy: No 'Access-Control-Allow-Origin' header`

**解决方案**：
在所有微应用的 `vercel.json` 中添加 CORS 头部配置（参见 5.2.1）

### 10.6 资源路径错误

**问题**：子应用 JS/CSS 文件 404

**原因**：publicPath 配置错误

**解决方案**：
- Webpack 应用检查 `output.publicPath`
- Vite 应用检查 `base` 配置
- 确保使用 `process.env.VERCEL_URL` 动态设置

### 10.7 构建超时

**问题**：构建过程超时失败

**解决方案**：
1. 优化 `node_modules` 安装：使用 `pnpm install --frozen-lockfile`
2. 减少不必要的依赖
3. 使用 `.vercelignore` 忽略不必要的文件：
   ```
   node_modules
   .git
   .vscode
   *.log
   ```

## 附录 C：环境变量管理脚本

创建 `scripts/sync-env.sh`：

```bash
#!/bin/bash

# 同步环境变量到所有 Vercel 项目

projects=(
  "erp-lite-shell"
  "erp-lite-user"
  "erp-lite-product"
  "erp-lite-order"
  "erp-lite-dashboard"
)

# API 基础 URL
api_url="https://api.erp-lite.com"

for project in "${projects[@]}"; do
  echo "配置 $project 环境变量..."

  vercel env rm VITE_API_BASE_URL production --yes --scope=$project || true
  echo $api_url | vercel env add VITE_API_BASE_URL production --scope=$project

  vercel env rm NODE_ENV production --yes --scope=$project || true
  echo "production" | vercel env add NODE_ENV production --scope=$project
done

echo "环境变量同步完成！"
```
