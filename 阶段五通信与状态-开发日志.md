```typescript
interface QiankunProps {
  // qiankun 全局状态管理
  globalState?: MicroAppStateActions;

  // 用户认证 token
  token?: string | null;

  // 用户信息
  userInfo?: {
    id: string;
    name: string;
    role: string;
    permissions: string[];
```

## 技术要点

### 1. React 应用的 Props 传递

使用 Context API：
- 创建 Context
- 提供 Provider 包裹 App
- 在组件中使用 useContext Hook

### 2. Vue 3 应用的 Props 传递

使用 Provide / Inject：
- 使用 `app.provide(key, value)` 提供数据
- 使用 `inject(key)` 注入数据
- 通过 InjectionKey 提供类型安全

### 3. 原生 JS 应用的 Props 传递

直接使用全局变量：
- 在 mount 函数中保存 props 到模块级变量
- 在渲染函数中读取变量使用

### 4. qiankun 全局状态管理

主应用：
- 使用 `initGlobalState` 初始化状态
- 使用 `setGlobalState` 更新状态
- 使用 `onGlobalStateChange` 监听变化

子应用：
- 通过 props 接收 globalState actions
- 调用 `setGlobalState` 通知主应用
- 调用 `onGlobalStateChange` 监听变化

## 遇到的问题

### 问题 1: Vue Composables 返回值类型

**现象**: Vue composables 的返回值需要使用 `.value` 访问

**解决**:
```typescript
// 错误写法
console.log('用户信息:', userInfo);

// 正确写法
console.log('用户信息:', userInfo.value);
```

### 问题 2: 原生 JS 应用 Props 传递时机

**现象**: 原生 JS 无法像框架一样自动传递 props

**解决**:
- 在 mount 函数中保存 props
- 在 initDashboard 函数中传递 props
- 在 renderDashboard 中从模块变量读取
